# Testing Guide

## Overview

The project has two levels of testing:

1. **Unit/Integration Tests** (`backend/tests/`) — pytest with async support, in-memory SQLite, mock LLM
2. **E2E Evaluation** (`scripts/run_eval.py`) — Runs all 12 test cases against the live API

## Quick Start

```bash
# Run all unit tests
uv run pytest backend/tests/ -v

# Run with coverage
uv run pytest backend/tests/ -v --cov=backend --cov-report=term-missing

# Run a specific test file
uv run pytest backend/tests/test_claim_pipeline.py -v

# Run a specific test
uv run pytest backend/tests/test_claim_pipeline.py::test_tc004_clean_consultation -v

# Run E2E eval (requires running server)
uv run python scripts/run_eval.py
```

## Test Configuration

Configured in `pyproject.toml`:

```toml
[tool.pytest.ini_options]
asyncio_mode = "auto"
testpaths = ["backend/tests"]
pythonpath = ["backend"]
```

- **asyncio_mode = "auto"** — All async tests run automatically without `@pytest.mark.asyncio`
- **pythonpath** — Adds `backend` to Python path for imports

## Test Fixtures

All fixtures are defined in `backend/tests/conftest.py`.

### Database Fixture

```python
@pytest.fixture
async def db_session():
    """In-memory SQLite database, fresh per test."""
    # Creates all tables, yields session, drops all tables after test
```

Each test gets a completely isolated database. No data leaks between tests.

### Policy Data Fixture

```python
@pytest.fixture
```

### Mock LLM Fixture

```python
@pytest.fixture(autouse=True)

    """Forces LLM_PROVIDER=mock for all tests."""
```

## Writing New Tests

### Test Naming Convention

```python
async def test_<descriptive_name>():
    """One-line description of what this test verifies."""
```

### Pattern: Pipeline Test

```python
async def test_new_scenario(db_session, sample_docs):
    """Test a specific claim scenario."""
    # Arrange
    from backend.orchestrator.engine import ClaimOrchestrator

    orchestrator = ClaimOrchestrator(db_session)

    # Act
    result = await orchestrator.process_claim(
        claim_id=1,
        member_id="EMP001",
        policy_id="PLUM_GHI_2024",
        claim_category="CONSULTATION",
        treatment_date=date(2026, 6, 4),
        claimed_amount=Decimal("1500"),
        documents=sample_docs,
    )

    # Assert
    assert result["decision"] == "APPROVED"
    assert result["approved_amount"] == Decimal("1350")  # After 10% co-pay
```

### Pattern: Policy Service Test

```python
async def test_policy_rule(db_session, policy_data):
    """Test a specific policy rule."""
    from backend.domain.policy.service import PolicyService

    service = PolicyService(policy_data)

    # Test waiting period
    result = service.is_within_waiting_period(
        join_date=date(2026, 4, 15),
        treatment_date=date(2026, 6, 4),
        diagnosis="Type 2 Diabetes Mellitus",
    )

    assert result.is_within_period is True
    assert result.eligible_from == date(2026, 7, 14)
```

### Pattern: Document Verification Test

```python
async def test_document_check(db_session):
    """Test document verification logic."""
    from backend.domain.documents.service import DocumentService

    service = DocumentService(db_session)

    docs = [
        {"file_id": "F001", "file_name": "rx.jpg", "actual_type": "PRESCRIPTION"},
    ]

    result = await service.verify_documents(
        claim_id=1,
        claim_category="CONSULTATION",
        policy_doc_reqs={"required": ["PRESCRIPTION", "HOSPITAL_BILL"]},
        uploaded_docs=docs,
    )

    assert result["is_valid"] is False
    assert any(e["error_type"] == "MISSING_REQUIRED" for e in result["errors"])
```

## Mock LLM Provider

The mock provider (`backend/providers/llm/mock_adapter.py`) returns deterministic responses for testing:

- `chat()` — Returns a generic acknowledgment
- `extract_structured()` — Returns pre-extracted data if provided in content, otherwise returns mock extraction
- `health_check()` — Always returns healthy

No API keys or network calls needed.

## E2E Evaluation Script

The eval script (`scripts/run_eval.py`) runs all 12 test cases against the live API:

```bash
# Start the server first
uv run uvicorn backend.main:app --host 0.0.0.0 --port 8000 &

# Seed the database
uv run python scripts/seed_data.py

# Run eval
uv run python scripts/run_eval.py
```

Output:

## Continuous Integration

To run tests in CI:

```bash
# Install dependencies
uv sync

# Run linting
uv run ruff check backend/

# Run tests with coverage
uv run pytest backend/tests/ -v --cov=backend --cov-report=xml

# Run E2E (requires services)
docker compose up -d
sleep 10
uv run python scripts/seed_data.py
uv run python scripts/run_eval.py
```
